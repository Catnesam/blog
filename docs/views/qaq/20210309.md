# axios

json-server使用：

安装后，在json文件上层

```shell
json-server --watch db.json
```



## 是什么

前端最流行的ajax请求库



**特点：**

- 基于xhr + promise的异步ajax请求
- 浏览器、node都可用
- 支持请求/响应拦截
- 支持请求取消
- 请求/响应数据转换
- 批量发多个请求

## 源码

### 源码结构

├── /dist/	#输出目录

├── /lib/	#源码目录

│ ├── /adapters/	#适配器，其实就是xhr\http两个，对应前端与node

│ │ ├── http.js

│ │ └── xhr.js

│ ├── /cancel/	#取消请求的功能

│ ├── /core/	#核心功能

│ │ ├── Axios.js	#核心主类

│ │ ├── dispatchRequest.js	#来调用适配器方法的函数

│ │ ├── InterceptorManager.js	#拦截器的管理函数

│ │ └── settle.js	#响应状态处理函数，改变Promise的对象

│ ├── /helpers/	#辅助方法

│ ├── axios.js	#接口暴露

│ ├── defaults.js	#默认配置

│ └── utils.js	#工具类

├── package.json

├── index.d.ts

└── index.js	#入口文件



### 总结

#### axios与Axios的联系

1. 从语法上来说，axios不是Axios的实例
2. 从功能来说，axios是Axios的实例
3. axios是Axios.prototype.request()函数bind()返回的对象
4. axios有Axios原型对象上的所有方法，有Axios对象上的所有属性

#### axios与instance实例

instance:  axios.js中createInstance（）创建出来的实例

1. 相同点：
   1. 都是能发任意请求的函数：request(config)
   2. 都有特定的方法：get\post\pust...
   3. 都有默认配置和拦截器属性：default、interceptors
2. 不同：
   1. 默认配置可能不同
   2. axios后面还添加了（create，CancelToken，all）等方法，instance没有
